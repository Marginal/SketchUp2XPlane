X-Plane Plugin für SketchUp

Übersicht

SketchUp ist ein freier (kostenfreier) 3D Objekt Editor für Windows und MacOS.

Dieses SketchUp Plugin exportiert Modelle im X-Plane v7 or v8 .obj Format.

Installation

Kopiere die Dateien SU2XPlane.html, SU2XPlane_DE.html und SU2XPlane.rb in deinen SketchUp Plugins Ordner:

  • Windows:
    SketchUp & Pro 6: C:\Program Files\Google\Google SketchUp 6\Plugins
    SketchUp 5: C:\Program Files\Google\Google SketchUp\Plugins
    SketchUp Pro 5: C:\Program Files\@Last Software\SketchUp 5\Plugins
  • Mac OS:
    Sketchup & Pro 6: /Library/Application Support/Google SketchUp 6/SketchUp/Plugins
    SketchUp 5: /Library/Application Support/Google SketchUp/Plugins
    SketchUp Pro 5: /Users/you/Library/Application Support/SketchUp 5/Plugins

SketchUp staren oder neu starten.

Du kannst nun die .zip Datei, die du herunter geladen hast, löschen. Die Hilfe-Datei, die du mit herunter geladen hast, kannst du unter Help → X-Plane in SketchUp aufrufen.

X-Plane-Szenerie-Objekte herstellen

Bevor du anfängst, kannst du Google Earth benutzen, um ein Satellitenfoto von der Gegend zu bekommen, an an der du arbeiten willst:

  • In Google Earth auf deine Gegend zoomen.
  • In SketchUp wähle Tools → Google Earth → Get Current View.

Stelle dein Modell in SketchUp aus Rechtecken und Kreisen her (jede Art vom Bögen - Arcs - wird ignoriert). Sichere die SketchUp .skp Datei in dem Aircraft- oder Scenery-Ordner, den du für X-Plane .obj Dateien vorgesehn hast.

In einem Bildbearbeitungsprogramm musst du eine einzige Textur-Datei wie eine "collage" von allen Texturen herstellen, die du in deinem Modell brauchst. Höhe und Breite deiner Textur Datei müssen eine Zweier-Potenz haben (z.B 128, 256, 512, 1024), aber sie muss kein Quadrat sein. Siehe …/Custom Scenery/KSBD Demo Area/KSBD_example.png als Basisbeispiel.

Sichere die Textur-Datei im 24bit oder 32bit PNG Format (d.h mit oder ohne "Alpha"-Kanal) in dem Aircraft- oder Scenery-Ordner, den du für deine X-Plane .obj Datei vorgesehen hast.

Benutze das Paint Bucket Tool, deine Textur-Datei auf jede Fläche deines Modells zu übertragen. Benutze Context menu → Texture → Position, um den korrekten Ausschnitt aus deiner Textur-Datei auf jeder Fläche zu positionieren (Klicke hier um ein Tutorial zu sehen).

Wenn du Textur auf einer gekrümmten Oberfläche positionieren willst, musst du zuerst View → Hidden Geometry auswählen. Dann selektiere eine einzelne Fläche auf dieser Oberfläche und positioniere nacheinander Textur auf jede Einzelfläche.

Wenn du möchtest, dass beide Seiten einer Fläche in X-Plane dargestellt werden, dann benutze das Paint Bucket Tool zweimal; je einmal für jede Seite der Fläche. Du kannst Window → Entity Info benutzen, um zu sehen, welche Textur(en) gerade den zwei Seiten der ausgewählten Fläche zugewiesen sind.

Benutze Tools → Highlight Untextured als Hilfe beim Texturieren. Das ändert nichts daran, wie dein Modell in X-Plane erscheinen wird, aber es macht irgendwelche Probleme beim Texturieren deutlich:

  1. Flächen ohne Textur sind rot gefärbt. Das hilft dir, diese Flächen zu erkennen.
  2. Bei Flächen, die nur auf einer Seite texturiert sind, ist die andere Seite unsichtbar (Durchsichtigkeit auf 0 gesetzt). Das hilft dir, wenn du die falsche Seite einer Fläche texturiert hast. Benutze Context menu → Reverse Faces auf der Fläche, wenn das geschehen ist

Attribute

Dieses Plugin erweitert das Context-Menü um drei Einträge. Diese können benutzt werden, wenn du eine Fläche aktiviert hast:

Hard
Bestimmt, ob ein Flugzeug landet auf/oder zerschellt an dieser Fläche in X-Plane. Flächen "hard" zu machen ist "teuer" (framerate), also benutze es sparsam. Vertikale "hard" Flächen wirken sich nicht so aus, wie du es erwarten würdest; wenn du ein Gebäude für Kollisionen geeignet machen willst, benutze "hard" für das Dach, nicht für die Seitenflächen.
(Bedenke, dass nur Flächen, die in Sketchup einfache Vierecke sind, in V7 Objekten "hard" sein können.
X-Plane Versionen vor 8.20 haben einen Fehler: .obj Dateien mit "harten" Flächen müssen in WorldMaker "object heading" 0 haben. Wenn du solche Objekte drehen willst, mach das in SketchUp und nicht in WorldMaker).
 
Ground
Bestimmt, ob eine Fläche mit "polygon offsetting" in X-Plane gerendert wird.
Benutze dies für Flächen, die flach auf dem Grund liegen, um Z-buffer thrashing in X-Plane zu verhindern. Benutze diesen Befehl nicht für andere Flächen..
(Berücksichtige, dass diese Einstellung nicht den gewünschten Effekt in einigen X-Plane Versionen vor 8.20 bringt).
 
Alpha
Dies ist ein Hinweis für X-Plane, dass diese Fläche mit einer teilweisen oder ganz transparenten oder durchscheinenden Textur versehen ist.
Benutze dies auf transparenten oder durchscheinenden Flächen, durch die du auf andere Flächen im selben Modell sehen willst. Drücke diesen Knopf nicht für opaque (normale) Flächen.

Export nach X-Plane

Wenn du es nicht schon getan hast, wähle File → Save As… und sichere die .skp Datei in dem Aircraft- oder Scenery-Ordner, den du für deine X-Plane .obj Datei vorgesehen hast

Wenn du ein Terrain-Foto von Google Earth geladen hattest, musst du es vor dem Export ausblenden:

  • Wähle Window → Layers.
  • Im Layers Panel vergewissere dich, dass "Google Earth Terrain" und "Google Earth Snapshot" beide abgewählt sind.
  • Wenn du das Terrain-Foto vor Beginn des Modellierens geladen hast, dann kannst du die Koordinaten, wo dein Modell in X-Plane platziert werden soll, in Window → Model Info Panel ablesen; wähle Location und drücke den Custom location… Knopf.

Dann wähle entweder:

File → Export X-Plane v7 Object
v7 Objekte können benutzt werden in X-Plane v7 oder v8.
Gekrümmte Oberflächen werden "stufig" in X-Plane erscheinen.
 
File → Export X-Plane v8 Object
v8 Objekte können nur in X-Plane v8.16 und später benutzt werden.
X-Plane wird gekrümmte Oberflächen korrekt darstellen.

Das Objekt wird im selben Ordner und mit demselben Namen wie die SketchUp Datei erzeugt, aber mit einer .obj Erweiterung.

Du köntest folgende Warnhinweise sehen:

Warning: You used multiple texture files. Using file name.
Du hast mehr als eine Textur-Datei benutzt, um die Flächen in seinem Modell zu texturieren.
X-Plane fordert, nur eine einzige Textur-Datei für alle deine Flächen zu benutzen.
 
Warning: n faces are untextured.
Du hast in deinem SketchUp Modell nicht auf alle Flächen Textur aufgetragen.
X-Plane wird untexturierte Flächen mit dem unteren linken Pixel deiner Textur-Datei färben (oder grau, wenn du keine Textur auf irgendeiner deiner Flächen platziert hast).

Wenn du eine der Warnungen siehst, wird X-Plane wahrscheinlich dein Modell nicht so darstellen, wie du es erwartet hast.

Einschränkungen

Konstruiere dein Sketchup Modell nur aus Rechtecken, Kreisen, Komponenten und Gruppierungen - alle anderen Sketchup Typen (einschließlich Bögen) werden nicht exportiert.

Dieses Plugin eignet sich nicht dazu, fortschrittliche X-Plane scenery primitives herzustellen. Das schließt ein:

  • Animationen
  • Attribute: z,B. Cockpit, Materialien etc
  • Level Of Detail
  • Lichter
  • Linien
  • Rauch
