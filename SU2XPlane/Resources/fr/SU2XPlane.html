X-Plane plugin pour SketchUp

Vue d'ensemble

SketchUp est un logiciel gratuit, éditeur d'objets 3D pour Windows et MacOS.

Ce plugin SketchUp importe et exporte les modèles vers X-Plane v9/v10 au format .obj.

Création d'objets X-Plane

Si vous créez un objet de décor alors, avant de commencer, vous pouvez obtenir une photo de la zone que vous êtes sur le point de modèlisé:

  • Choisissez Fenêtre → Infos sur le modèle → Géoposition → Ajouter un emplacement…

Créez votre modèle dans SketchUp avec des rectangles, des cercles et des arcs. Enregistrez le fichier SketchUp .skp dans le dossier des avions ou de paysage où vous voulez que le fichier X-Plane .obj se retrouve.

Dans une application d'édition d'image, créer un fichier de texture qui ressemble à un «collage» de toutes les textures que vous avez besoin dans votre modèle. La hauteur et la largeur du fichier texture doit être en puissance de deux (par exemple 128, 256, 512, 1024, 2048) mais pas spécialement carré. Enregistrez le fichier texture en 32 bits ou 24 bits au format PNG (c'est à dire avec ou sans canal «Alpha») dans le dossier avion ou paysage où vous voulez que le fichier X-Plane .obj se retrouve.

Dans SketchUp, créez un nouveau «Matière» de votre fichier de texture:

  • Choisissez Fenêtre → Matières
  • Windows: Appuyez sur le bouton Créer une matière…, puis cochez Utiliser l'image de texture et sélectionnez votre fichier de texture
  • Mac OS: Choisissez l'onglet Palettes de textures, puis choisissez Couleurs dans le modèle dans le menu déroulant. Au bas de la boîte de dialogue choisissez Couleur → Nouvelle Texture… et sélectionnez votre fichier de texture
  • Spécifiez la largeur et la hauteur approximative

Utilisez l'outil Colorier pour appliquer votre texture d'un fichier à chaque face de votre modèle. Utilisez le menu contextuel → Texture → Position de cartographie la partie correcte de ce fichier de texture à chaque face. (Si vous ne trouvez pas la «Texture» entrée dans le menu contextuel, choisissez le menu contextuel → Inverser les faces et réessayer).

Si vous voulez que les deux côtés d'une face soient affichées dans X-Plane, appliquer l'outil Colorier à deux reprises, une fois de chaque côté dela face. Vous pouvez utiliser la Fenêtre → Infos sur l'entité panneau pour voir quelles textures sont actuellement affectées aux deux côtés de la face sélectionnée.

Vous pouvez utiliser Outils → Mettez en surbrillance non texturés pour vous aider à texturer. Cela ne va pas changer la façon dont votre modèle apparaîtra dans X-Plane, mais elle mettra en évidence, dans SketchUp, des problèmes avec votre texture:

  • Faces non texturés colorées en rouge. Cela vous aide à identifier ces faces.
  • Pour les faces qui sont texturées sur un seul côté, l'autre côté est caché (régler l'opacité à 0). Cela vous permet de voir si vous avez texturé du mauvais côté toutes les faces. Utilisez le menu contextuel → Inverser les faces sur la face si vous aviez cela.

Attributs

Ce plugin ajoute six entrées au menu contextuel. Ceux-ci peuvent être utilisés lorsque vous avez des face sélectionnés:

Hard
Contrôle si l'aéronef peut atterrir sur et / ou entrer en collision avec une face dans X-Plane.
Faire faces “hard” est couteux, donc utiliser cette option avec parcimonie. Vertical “hard” des faces ne fonctionnent pas comme on pouvait s'y attendre, si vous voulez faire un bâtiment «collisable» il suffit d'utiliser ce bouton pour le toit mais pas sur les côtés du bâtiment.
 
Hard Deck
Comme ci-dessus, mais aéronef peut aussi voler sous cette face dans X-Plane.
 
Ground
Contrôle si la face est rendue avec «polygon offsetting» dans X-Plane et, dans X-Plane v10, est drapée sur le terrain.
Utilisez ce sur les faces qui se trouvent à plat sur le sol pour éviter de raclée (Z-buffer) sous la surface d'X-Plane. Ne pas appuyer sur ce bouton pour les autres faces.
 
Alpha
Il s'agit d'une allusion à X-Plane pour que la face soit texturée avec une texture partiellement ou totalement transparente ou translucide.
Utilisez-le sur toutes les faces transparentes ou translucides à travers lesquelles vous voulez voir les autres faces du même modèle. Ne pas appuyer sur ce bouton pour les faces opaques (normale).
 
Shiny
Contrôle si la face est rendue aussi brillant dans X-Plane.
 
Animation
Voir ci-dessous.

Lumières

Utilisez l'outil Texte pour créer des lumières:

  • Tout d'abord, utilisez l'outil Mètre pour créer un «Point de guidage» à l'endroit où vous voulez que la lumière apparaisse.
  • Choisissez Outils → Texte.
  • Sélectionnez le point que vous venez de créer, puis faites glisser votre souris (ce qui crée une flèche pointant vers votre point de repère), puis cliquez dessus.
  • Tapez le nom de l'un des types de lumière documentés ici, c'est-à-dire LIGHT_NAMED, LIGHT_CUSTOM, LIGHT_PARAM, LIGHT_SPILL_CUSTOM, smoke_black ou smoke_white.
  • Tapez également les paramètres requis, sauf pour les paramètres <x> <y> <z>, dont les valeurs seront fournies par le plugin.
    Remarque : Pour les lumières de type LIGHT_NAMED et LIGHT_PARAM le fichier X-Plane/Resources/bitmaps/world/lites/lights.txt répertorie les valeurs utilisables pour le paramètre <name>.

Il existe quelques créations de modèles de lumières ici.

Animations

X-Plane prend en charge deux types d'animations, le déplacement/rotation et masquer/afficher. Ces deux types d'animation sont contrôlés à partir d'une seule boîte de dialogue dans SketchUp:

  • Sélectionnez l'objet(s) que vous souhaitez animer.
  • Choisissez le menu contextuel → X-Plane → Animation…

SketchUp groupe le ou les objet(s) que vous avez sélectionné(s) dans un "Composant" (affiché comme une boîte rectangulaire) et affiche un dialogue d'«X-Plane Animation», décrit ci-dessous.

Il y a quelques exemples de création de modèles d'animations ici.

“DataRefs”

Vous pouvez utiliser le simulateur “DataRefs” de type “int”, “float” ou “double” énumérés ici pour créer n'importe quel type d'animation.

  • Les Datarefs de type “int” contiennent des valeurs en nombres entiers, représentant généralement oui/non ou marche/arrêt, et ont donc tendance à être adaptés pour masquer/afficher des types d'animations.
  • Les Datarefs de type “float” et “double” contiennent des valeurs fractionnaires et peuvent convenir pour les deux types d'animation.
  • Les Datarefs dont le type est suivi de «[n]» sont des “tableaux” qui contiennent plusieurs valeurs, dont la signification dépend de la Dataref. Vous devrez choisir parmi les n valeurs que vous souhaitez utiliser pour votre animation.
    par exemple sim/flightmodel2/doors/angle_now_deg est mentionné comme étant de type «float[doors]» (portes) - pour cette Dataref, chaque valeur du tableau représente l'angle de rotation d'une porte de train d'atterrissage.

Les Datarefs sont regroupés logiquement par nom. Quelques exemples:

sim/aircraft/...
Valeurs obtenues à partir du fichier .acf de l'avion que la personne pilote.
 
sim/cockpit/... & sim/cockpit2/...
L'état des instruments de bord et des commutateurs dans l'avion que la personne pilote.
(sim/cockpit/... est obsolète, utilisez sim/cockpit2/... pour les nouveaux modèles).
 
sim/flightmodel/... & sim/flightmodel2/...
L'état des moteurs, des ailes, du train d'atterrissage, etc de l'avion que la personne pilote.
(sim/flightmodel/... est obsolète, utilisez sim/flightmodel2/... pour les nouveaux modèles).
 
sim/graphics/...
Les différents paramètres de rendu et de l'état du monde sont sans relation avec l'avion que la personne pilote.
  • sim/graphics/scenery/percent_lights_on est utile si vous souhaitez animer quelque chose qui change à l'aube et au crépuscule.
 
sim/time/...
Les valeurs du temps sont sans relation avec l'avion que la personne pilote.
  • sim/time/total_running_time_sec est utile si vous souhaitez créer une animation qui se répète indéfiniment puisqu'elle compte que quelques secondes.
  • sim/time/local_time_sec est utile si vous souhaitez animer quelque chose qui dépend de l'heure du jour.
  • sim/time/local_date_days est utile si vous souhaitez animer quelque chose qui dépend de la période de l'année.

Vous pouvez utiliser le plougin DataRefEditor de X-Plane pour surveiller la valeur d'une de ces Datarefs dans X-Plane.

Déplacement / Rotation

Si vous voulez que votre ou vos objet(s) tournent (et ne se déplacent pas seulement d'un côté à l'autre), vous devez spécifier le point autour duquel la rotation doit se produire - le “centre de rotation”. Ce point est précisé dans SketchUp en utilisant l'«origine» des «axes» du Composant:

  • Tout d'abord, utilisez l'outil Mètre pour créer un «Point de guidage» au centre de la rotation.
  • Cliquez sur le Composant et choisissez le menu contextuel → Modifier les axes.
    SketchUp affiche l'emplacement actuel et l'orientation des axes du Composant.
  • SketchUp montre également de nouveaux axes sur le curseur de la souris. Déplacez-le sur le point de guidage que vous venez de créer et cliquez pour définir l'origine des nouveaux axes.
  • Cliquez encore deux fois pour définir l'orientation des axes rouge et vert du Composant. (C'est généralement moins déroutant si vous les orientez parallèlement aux axes globaux rouge et vert).

Les animations Déplacement/Rotation utilisent le concept d'une “Image-clé” (Keyframe) qui est une position et une valeur dataref correspondante. Vous devez spécifier deux ou plusieurs images-clés et X-Plane interpolera ou extrapolera pour créer une transition en douceur entre les positions basées sur la valeur actuelle de la dataref (c'est ce qu'on appelle “tweening”).

Dans la boîte de dialogue «X-Plane Animation», précisez:

  • Le nom de la Dataref qui conduit l'animation - l'une des datarefs de type “int”, “float” ou “double” énumérées ici - et, si la Dataref est répertoriée avec «[n]» après son type, le nombre de l'index de tableau.
  • Les valeurs de Dataref pour chaque Image-clé.
    Vous devez spécifier des valeurs de Dataref en ordre croissant. (C'est OK pour utiliser la même valeur de Dataref pour des images-clés adjacentes - ce qui peut être utile si vous voulez que le ou les objet(s) animé(s) saute(nt) d'une position à une autre).
  • La position du ou des objet(s) animé(s) de chaque image-clé; utilisez les outils Déplacement et Faire pivoter de SketchUp pour changer la position du Composante qu contient vos objets, puis appuyez sur une des touches «Définir» pour spécifier cette position.
    Lorsque vous utilisez l'outil Faire pivoter, vous êtes susceptibles d'obtenir les résultats que vous attendez dans X-Plane, sauf si vous tournez le Composant autour de l'origine de ses axes précis; utilisez alors le Point de guidage que vous avez créé ci-dessus. Évitez les rotations de 180° ou plus entre les images-clés - SketchUp ou ce plugin vont souvent au plus court.
  • Une valeur de «Boucle», le cas échéant.
    Lorsque vous spécifiez une valeur de Boucle, la valeur de la Dataref dans X-Plane est divisée par la valeur de Boucle, et le reste est utilisé pour conduire l'animation.

Vous pouvez utiliser les boutons «+» et «−» pour ajouter ou supprimer des images-clés, si nécessaire. Utilisez les boutons de «Rappel» pour rappeler la position associée à une image-clé.

Faites glisser le curseur sur «Aperçu» pour visualiser la façon dont le ou les objet(s) animé(s) se déplace(nt) entre les positions des images-clés dans X-Plane. Le nombre de la droite du curseur est la valeur Dataref qui correspond à le ou les objet(s) animé(s) de position actuelle. Si le curseur Aperçu est désactivé, cela signifie que vous n'avez pas spécifié correctement certains des renseignements requis dans cette boîte de dialogue. Si votre ou vos objet(s) ne s'anime(nt) pas comme vous l'espériez que votre revue, alors revoyez vos axes de Composant, les positions des images-clés et/ou des valeurs dataref.

Masquage / Affichage

Vous pouvez choisir de masquer des objets sur la base d'une valeur de Dataref dans X-Plane. Dans la boîte de dialogue “X-Plane Animation”, appuyez sur le bouton du bas «+» et précisez:

  • Le nom de la Dataref qui conduit l'animation - l'une des datarefs de type “int”, “float” ou “double” énumérées ici - et, si la Dataref est répertoriée avec «[n]» après son type, le nombre de l'index de tableau.
  • La plage des valeurs de la Dataref qui devrait cacher votre ou vos objet(s).

Vous pouvez spécifier plusieurs conditions - utiliser les boutons «+» et «−» pour ajouter ou supprimer des conditions. Vous pouvez également spécifier qu'un objet précédemment caché peut être affiché. La dernière condition active “gagne” (c'est-à-dire que les conditions ne sont pas “empilées”), l'ordre des conditions est significatif.

Animations complexes

Les animations complexes peuvent être modélisées à l'aide d'une hiérarchie de Composants. Souvent, la nature physique de l'objet que vous modélisez dictera la hiérarchie, par exemple le compensateur d'aileron d'un avion est attaché à son aileron (et se déplace quand l'aileron tourne) il devrait donc être modélisé comme une animation “enfant” de l'animation aileron. Pour créer une animation enfant:

  • Double-cliquez sur l'animation existante (“parente”) pour l'ouvrir.
  • Sélectionnez le ou les objet(s) que vous souhaitez animer. (Cela peut être l'ensemble du ou des objet(s) dans le Composant ou un sous-ensemble - dans notre exemple, nous choisissons simplement le compensateur, pas l'aileron entier).
  • Choisissez le menu contextuel → X-Plane → Animation… et procédez comme décrit ci-dessus.

Utilisez la boîte de dialogue «Structure» (Fenêtre → Structure) pour visualiser la hiérarchie du Composant. Vous pouvez modifier les relations parent/enfant entre les Composants en faisant glisser les entrées haut et bas de cette fenêtre.

Exportation vers X-Plane

Si vous ne l'avez pas encore fait, choisissez Fichier → Enregistrer sous… et enregistrez le fichier .skp dans le dossier avion ou le dossier paysage où vous voulez que le fichier X-Plane .obj se retrouve.

Si vous avez chargé une photo du terrain de Google Earth, il vous faut le cacher avant de les exporter:

  • Choisissez Fenêtre → Calques.
  • Dans le panneau Calques veiller à ce que «Google Earth terrain» et «Google Earth Snapshot» soient décochés.
  • Si vous avez chargé la photo du terrain avant de commencer la modélisation, vous pouvez lire les coordonnées où votre modèle doit être placé dans OverlayEditor ou WED à partir du panneau Fenêtre → Infos sur le modèle → Géoposition.

Ensuite:

  • Choisissez Fichier → Export X-Plane Object

Vous pouvez avoir les avertissements suivants:

Attention: Vous avez utilisé plusieurs fichiers de texture. Fichier utilisé: nom.
Vous avez utilisé plus d'un fichier de texture pour les faces de votre modèle.
X-Plane vous oblige à n'utiliser qu'un seul fichier de texture pour l'ensemble des faces de votre modèle.
 
Avertissement: n faces n'ont pas de texture.
Vous n'avez pas appliqué de texture à l'ensemble des faces de votre modèle SketchUp.
X-Plane sera de la couleur des faces non texturées avec le pixel en bas à gauche de votre fichier de texture (ou gris si vous n'avez pas appliqué une texture à l'une des faces).

Si vous voyez l'un de ces avertissements, X-Plane ne sera probablement pas capable d'afficher votre modèle comme vous le souhaitiez.

Limites

Construisez votre modèle SketchUp avec des rectangles, des cercles, des arcs, les Composants et Groupes seulement - tous les autres types de SketchUp ne seront pas exportés.

Ce plugin ne vous permet pas de créer des primitives spécifiques dans X-Plane, comme:

  • Des attributs: cockpit par exemple, matériaux, etc
  • Des niveaux de détail
  • Des lignes.
